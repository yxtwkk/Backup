事务处理 Transaction Processing

OLTP = Online Transaction Processing 存在大量并发用户，少量的随机读写操作

Transaction事务

（1）一个事务包含多个操作：select/insert/delete/update etc

（2）每个SQL语句可以是一个事务

（3）具有特殊的语句，表示事务Transaction的开始／成功结束／异常结束等

ACID:事务的四个性质

（1）Atomicity 原子性：事务要么全部执行，要么全不执行 ALL or Nothing

（2）Consistency 一致性： 从一个正确状态转换到另一个正确状态✅

（3）Isolation 隔离性：每个事务与其他并发事务互不影响

（4）Durability 持久性：事务提交以后，持久有效，crash也不消失

思考：同一个数据元素被并发访问 会产生什么问题？

=> 数据竞争 Data Race：当两个并发访问都是写，或者一个读，一个写

必然存在一个顺序，按照这个顺序依次串行执行这些Transactions，得到的结果与并行执行时相同
